Building Teamwork: Mixed Reality Game for Developing Trust and Communication

Description: A mixed reality VR application task digitizing legos from a model in the real world, to a VR application, by the use of image processing and socket streaming.

Tech Used/Competences Gained: Python for image processing, Unity/C# for scene creation of VR application, socket streaming for communication between Python and Unity/C#.

Program briefly explained:

  1. The VR user has two Lego "statues" in front of them in a puzzle room and needs them to match. However they do not have any legos in possession, so they have to communicate with their partner.
  2. The user not in VR is set in front of a lego plane and has to build the figure which the VR player tells them too. Since they are not in VR, they can only rely on verbal information given by the VR player.
  3. Once both figures match, a door in the VR application opens and the VR player can progress.

FoodFighters - Improving Memory Retention of Food Items through a Mobile Serious Game

Description: An application made to reduce food waste via memory retention and gamification techniques.

Tech Used/Competences Gained: Making applications in Android Studio. Creating "character" customizer and saving user made creations.

Program briefly explained:

  1. Take a picture of a food-fighter you wish to create.
  2. Crop said photo by dragging finger around the entire food item. Confirm when you are happy with the crop.
  3. Customize your food fighter with options for hands/feet/hair/eyes/mouth.
  4. When customization is done, give the character a name.
  5. Go into a 1 on 1 battle with a list of pre-set characters in the application.

Bouncing Ball Traversal: A Target-Based Travel Technique For Vertical Travel

Description: Alternative teleportation method for virtual reality. Explored in association with the course "Spatial User Interfaces", but later implemented into a main project exploring user profiling in VR with the use of machine learning.

Tech Used/Competences Gained: Unity/C#. As well gathering biometric data from VR applications for ML purposes.

Program briefly explained:

  1. The user picks up the ball and gauges what location they want to go to.
  2. When the user has found their location. They have an option to "toggle" on the ball how many bounces it should do before teleportation.
  3. The user throws the ball and gets teleported to the location next time the ball hits a ground plane. In which the process can be repeated.

Disconnected: An Interactive Edutainment Game Concerning Influence And Use Of Social Media

Description: Edutainment game tackling the influence of social media. Made in association with the course "Narratives in Interactive Systems". Has branching paths and ending.

Tech Used/Competences Gained: Unity/C# and INK dialogue system. Creation of evolving systems, such as "feed"

Program briefly explained:

  1. the user goes through a 3 day cycle where they day 1 talk to their offline friend. While just getting introduced to the internet. The second day they find an online friend and on the last day they have to juggle both relationships at the same time.
  2. The options a user has is either to scroll their social media feed. Which is color coded based on what type of post it is, talking to either of their friends over messenger apps or going to their personal profile.
